# Proposal: Edit Sent Messages

## Why

Пользователи Todo List не могут редактировать текст созданных сообщений после отправки. Это создаёт проблемы при опечатках, изменениях планов или необходимости уточнить информацию. Текущее решение — удаление и пересоздание сообщения — неудобно и разрушает контекст.

Backend уже поддерживает обновление текста через существующий PUT endpoint, но UI не предоставляет интерфейс для редактирования.

## What Changes

Добавить возможность редактировать текст существующих todo-сообщений без удаления и пересоздания:

- **Frontend**: Добавить UI-механизм редактирования текста (кнопка Edit + modal dialog)
- **Backend**: Использовать существующий PUT `/api/todos/{id}` для сохранения изменений
- **UX**: Optimistic UI update с rollback при ошибке
- **Validation**: Базовая валидация ввода (maxlength, required)

Изменение не затрагивает:
- Структуру БД (схема остаётся прежней)
- Логику создания/удаления todos
- Механизм переключения completed

## Capabilities

- `todo-text-editing`: Возможность редактировать текст существующего todo через UI

## Impact

**Затрагиваемые компоненты:**
- Frontend: `app/page.tsx` (новый компонент EditModal, edit flow)
- Backend: нет изменений (PUT endpoint уже готов)
- Data model: нет изменений

**Риски:**
- Низкий риск: backend уже протестирован и работает для toggle
- UX риск: modal dialog может быть менее удобен чем inline editing (можно улучшить позже)

**Зависимости:**
- Нет зависимостей от других изменений
- Можно реализовать независимо

## Success Criteria

1. Пользователь может отредактировать текст todo через кнопку Edit
2. Редактирование сохраняется в БД через существующий PUT endpoint
3. UI показывает обновлённый текст после сохранения
4. Ошибка API обрабатывается с rollback UI и сообщением об ошибке
5. Валидация ввода предотвращает пустые тексты и слишком длинные сообщения
