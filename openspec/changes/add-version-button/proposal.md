# Proposal: Add Version Button

## Why
В текущем приложении отсутствует способ отображения информации о версии для пользователей и разработчиков. Это затрудняет:
- Идентификацию развернутой версии при отладке
- Проверку соответствия версий frontend и backend
- Информирование пользователей о текущей версии приложения

Добавление кнопки версии позволит пользователям быстро получить информацию о версии приложения без необходимости заглядывать в package.json или pom.xml.

## What Changes
- Добавится UI-кнопка "Version" на фронтенд
- Создастся новый backend endpoint `GET /api/version` для возврата версии backend
- При клике на кнопку будет отображаться информация о версии backend (и опционально frontend)
- Кнопка будет интегрирована в существующий UI Todo List приложения

## Capabilities
- `ui-version-button`: UI-компонент кнопки с логикой отображения версии
- `api-version-endpoint`: Backend endpoint для получения версии backend
- `version-display`: Логика форматирования и отображения информации о версии

## Impact
- **Frontend**: добавится новый UI-элемент и state для хранения версии
- **Backend**: добавится один REST endpoint
- **Пользователи**: получат доступ к информации о версии через UI
- **Разработчики**: упростится идентификация развернутой версии

## Success Criteria
- Backend endpoint `GET /api/version` возвращает JSON с версией backend
- Кнопка "Version" отображается на UI
- При клике на кнопку показывается версия backend (0.0.1-SNAPSHOT)
- Проект собирается без ошибок: `cd backend && ./gradlew build` и `cd frontend && npm run build`
