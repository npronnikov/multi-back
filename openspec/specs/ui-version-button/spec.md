# UI Version Button Specification

## Purpose
Добавляет UI-компонент кнопки для отображения информации о версии приложения.

## Requirements

#### Requirement: Кнопка Version на UI
Фронтенд должен отображать кнопку для доступа к информации о версии приложения.

##### Scenario: Отображение кнопки
- **WHEN** пользователь открывает приложение Todo List
- **THEN** на странице отображается кнопка с текстом "Version"
- **AND** кнопка расположена в header рядом с заголовком "Todo List"
- **AND** кнопка имеет стилизацию, соответствующую общему стилю приложения (secondary или ghost button)

##### Scenario: Клик по кнопке
- **WHEN** пользователь кликает на кнопку "Version"
- **THEN** инициируется запрос к backend endpoint `GET /api/version`
- **AND** после успешного получения ответа отображается информация о версии
- **AND** при ошибке запроса отображается сообщение об ошибке

#### Requirement: Отображение версии
После успешного получения версии она должна быть отображена пользователю.

##### Scenario: Успешное отображение версии
- **WHEN** backend успешно вернул версию
- **THEN** пользователю показывается alert/popup с текстом:
  ```
  Backend version: {version}
  Timestamp: {timestamp}
  ```
- **AND** значения подставляются из ответа backend

##### Scenario: Ошибка при получении версии
- **WHEN** запрос к backend завершился с ошибкой
- **THEN** пользователю показывается alert/popup с текстом:
  ```
  Failed to load version information.
  ```
- **AND** консоль браузера содержит лог ошибки
