# Version Display Specification

## Purpose
Определяет логику форматирования и отображения информации о версии приложения.

## Requirements

#### Requirement: Форматирование версии
Информация о версии должна быть корректно форматирована перед отображением.

##### Scenario: Форматирование timestamp
- **WHEN** получен ответ от backend с полем `timestamp` в формате ISO-8601
- **THEN** timestamp отображается в читаемом формате для пользователя
- **AND** при ошибке парсинга используется исходное значение из ответа

##### Scenario: Отсутствие полей в ответе
- **WHEN** в ответе backend отсутствуют обязательные поля (`version` или `timestamp`)
- **THEN** отображается сообщение об ошибке "Invalid version data received"
- **AND** консоль содержит предупреждение о некорректных данных

#### Requirement: Хранение состояния версии
Frontend должен хранить полученную информацию о версии в состоянии компонента.

##### Scenario: Сохранение версии в state
- **WHEN** backend успешно вернул версию
- **THEN** данные версии сохраняются в state компонента
- **AND** последующие клики на кнопку "Version" используют кэшированные данные
- **AND** состояние ошибки (если была) очищается

##### Scenario: Обработка состояния загрузки
- **WHEN** запрос к версии выполняется
- **THEN** кнопка "Version" визуально показывает состояние загрузки (disabled или индикатор)
- **AND** повторные клики во время загрузки игнорируются
