# Version YAML Specification

## Purpose
Добавляет возможность сохранения информации о версии в YAML-конфигурацию. Это delta-spec определяет поведение при записи версии в YAML-файл.

## Requirements

### Requirement: Save Version to YAML
Система должна сохранять версию приложения в YAML-файл при запросе версии.

#### Scenario: Create YAML on first request
- **WHEN** клиент отправляет первый `GET /api/version` и YAML-файл не существует
- **THEN** система создаёт файл `version.yml` в `src/main/resources/`
- **AND** файл содержит запись `version: <версия приложения>`
- **AND** клиент получает ответ с версией

#### Scenario: Update existing YAML
- **WHEN** клиент отправляет `GET /api/version` и YAML-файл уже существует
- **THEN** система обновляет файл `version.yml` текущей версией
- **AND** структура файла остаётся корректной YAML
- **AND** клиент получает ответ с версией

#### Scenario: YAML format
- **WHEN** версия записана в YAML-файл
- **THEN** файл содержит только ключ `version` со строковым значением
- **AND** формат соответствует YAML спецификации
- **AND** файл может быть прочитан стандартными YAML-парсерами

#### Scenario: Concurrent writes
- **WHEN** множество клиентов одновременно запрашивают `/api/version`
- **THEN** только один процесс пишет в YAML-файл
- **AND** файл остаётся в корректном состоянии
- **AND** все запросы успешно завершаются

#### Scenario: File system errors
- **WHEN** система не может создать или обновить YAML-файл (например, нет прав доступа)
- **THEN** API всё равно возвращает версию в ответе
- **AND** ошибка логируется для диагностики
- **AND** HTTP-код ответа остаётся 200 OK
