# Архитектура репозитория (моя редакция)

## Заголовок
Фронтенд-приложение на Next.js

## Repo alias
main

## Назначение репозитория
Фронтенд-приложение multi-front — это веб-интерфейс для управления списком задач (Todo List). Приложение предоставляет пользовательский UI для создания, редактирования и удаления задач, взаимодействуя с backend API через RESTful HTTP-запросы.

## Наблюдения по архитектуре

1. **Next.js App Router с клиентскими компонентами**. Приложение использует современный Next.js 16.2.10 с App Router (`app/` директория). Основной компонент `page.tsx` помечен директивой `"use client"`, что означает использование React Client Components с хуками `useState`, `useEffect` для управления состоянием на клиенте.

2. **Tailwind CSS для стилизации**. Используется Tailwind CSS v4 через `@tailwindcss/postcss`, что обеспечивает utility-first подход к стилизации. В коде видно использование dark mode (`dark:bg-black`, `dark:text-zinc-50`) и responsive дизайна.

3. **Интеграция с backend API**. Приложение взаимодействует с backend по REST API (по умолчанию `http://localhost:8081/api/todos`). Реализованы все CRUD операции: GET для загрузки задач, POST для создания, PUT для обновления статуса, DELETE для удаления. Есть обработка ошибок и оптимистичные обновления UI.

4. **TypeScript и строгая типизация**. Используется TypeScript с явно определённым интерфейсом `Todo`. Приложение следует современным практикам React с типизацией пропсов и состояний.

5. **Конфигурация через переменные окружения**. URL backend API вынесен в `NEXT_PUBLIC_API_URL` с fallback на `http://localhost:8081`, что позволяет легко менять окружение (dev/prod).
