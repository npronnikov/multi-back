# Архитектура: main

**Repo alias:** `main`

## Назначение репозитория

`multi-back` — это бэкенд-сервис для приложения «To-Do List» (список задач), построенный на Spring Boot 4.1.0 / Java 21. Обеспечивает REST API для CRUD-операций над задачами, использует встроенную файловую базу данных H2 и настроен на приём CORS-запросов от фронтенд-клиента на порту 8082.

## Наблюдения по архитектуре

1. **Упрощённая слоистая архитектура без Service-слоя.** Контроллер `TodoController` обращается напрямую к `TodoRepository` через внедрение зависимости в конструкторе. Для небольшого CRUD-приложения это допустимо, но при росте логики потребуется выделение промежуточного сервисного слоя.

2. **Единый Maven-модуль с плоской структурой пакетов.** Весь код находится в одном пакете `com.example.demo` (5 классов: `DemoApplication`, `WebConfig`, `Todo`, `TodoRepository`, `TodoController`). Модульность отсутствует, что усложнит разделение ответственности по мере роста кодовой базы.

3. **Spring Data REST подключён, но экспорт отключён.** В `pom.xml` присутствует `spring-boot-starter-data-rest`, однако `TodoRepository` аннотирован `@RepositoryRestResource(exported = false)`. Маршрутизация полностью реализована вручную в контроллере — вероятно, намеренное решение для полного контроля над HTTP-статусами и форматом ответов.

4. **Файловый режим H2 с автообновлением схемы.** База данных работает в файловом режиме (`jdbc:h2:file:./data/tododb`), данные сохраняются между перезапусками. Hibernate `ddl-auto=update` автоматически создаёт и обновляет схему, что удобно для разработки, но рискованно для production. Веб-консоль H2 доступна по `/h2-console`.

5. **Минимальное тестирование.** Единственный тест `DemoApplicationTests.contextLoads()` проверяет подъём Spring-контекста. Отсутствуют модульные тесты для контроллера, репозитория или бизнес-логики, а также интеграционные тесты для REST-эндпоинтов.
